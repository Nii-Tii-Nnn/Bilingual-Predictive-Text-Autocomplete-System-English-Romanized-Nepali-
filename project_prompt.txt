REQUIRED DATA STRUCTURES & ALGORITHMS
==================================================

Implement and explain:

1. TRIE
   Used for:
   - Prefix search
   - Fast autocomplete
   - Word storage

   Operations:
   - insert(word)
   - search(word)
   - delete(word)
   - starts_with(prefix)
   - get_suggestions(prefix)

2. HASH MAP / DICTIONARY
   Used for:
   - Word frequency tracking
   - User learning data
   - Analytics

   Example:
   {
       "hello": 120,
       "sanchai": 75,
       "computer": 45
   }

3. HEAP / PRIORITY QUEUE
   Used for:
   - Top-K prediction ranking
   - Efficient retrieval of most relevant suggestions

4. SORTING
   Used as fallback ranking mechanism.

   Compare:
   - Timsort (Python sort)
   - Heap-based ranking

5. FILE HANDLING
   Files:
   - dictionary.txt
   - frequency.json
   - history.json

   Implement:
   - Loading
   - Saving
   - Updating
   - Exporting

6. OPTIONAL ADVANCED ALGORITHMS

   a) Levenshtein Distance
      For typo correction.

      Examples:
      sanchaii -> sanchai
      compter -> computer
      enginering -> engineering

   b) N-Gram Prediction
      For next-word prediction.

      Examples:
      How are -> you
      k cha -> ta
